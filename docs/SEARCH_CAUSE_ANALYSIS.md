# 地図検索で結果が出ない原因とAPI連携について

## 1. 検索結果が0件になる原因

### 仕組み

店舗検索は**自社DBに登録された店舗**のみが対象です。店舗は特定の緯度・経度に固定されており、**検索地点から半径○km以内**にある店舗だけがヒットします。

### 0件になる主な理由

| 原因 | 説明 |
|------|------|
| **検索地点が店舗から遠い** | ピンを立てた場所から、最も近い店舗までが検索半径（例: 5km）を超えている |
| **店舗が登録されていない地域** | 北海道、沖縄、地方都市など、まだ店舗データがない地域 |
| **検索半径が狭い** | 半径3kmだと、駅から少し離れただけでヒットしない |

### 現在の店舗カバー範囲

自社DBには以下の主要駅周辺に店舗を登録しています（各エリア1〜数点）:

- 東京: 新宿、渋谷、池袋、横浜
- 関西: 大阪・梅田
- 中部: 名古屋・栄、名古屋駅
- 九州: 福岡・博多

※ 同じ都市内でも、駅から5km以上離れた場所では0件になる場合があります。

## 2. Google Places API 連携について

### 連携すると何が変わるか

| 項目 | 未連携（現状） | 連携後 |
|------|----------------|--------|
| 店舗の取得元 | 自社DBのみ | 自社DB + **Places API** |
| 検索範囲 | 登録済み座標の周辺のみ | **全国どこでも**対応チェーン店を発見可能 |
| 食事データ（メニュー・カロリー・価格） | 自社DB | **自社DB**（変わらず） |

### 重要なポイント

**Places API が提供するのは「店舗の名前・位置」だけです。**

メニュー・カロリー・価格は自社DBにあり、Places API で見つけた店舗の**名前**が対応チェーン（大戸屋、すき家、マクドナルド等）にマッチした場合のみ、自社のメニューデータと紐づけて表示できます。

- **連携すれば**: 対応チェーンの店舗が全国で見つかりやすくなる
- **連携しても**: 非対応チェーン店のメニュー・カロリーは表示できない（データがないため）

### 連携方法

`.env` に以下を設定:

```
GOOGLE_PLACES_API_KEY=your_api_key_here
```

[Google Cloud Console](https://console.cloud.google.com/) で Places API を有効化し、APIキーを発行してください。

## 3. 改善策（実装済み・予定）

- 検索半径を 5km に拡大（より広範囲でヒット）
- 0件時に「検索可能エリアが限られています」と案内
- Places API 連携で全国の対応チェーン店を検索可能に
